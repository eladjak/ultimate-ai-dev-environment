# 🇮🇱 תיקון בעיית העברית ב-Windows Terminal

## הבעיה

כאשר עובדים עם Claude Code CLI בטרמינל ב-Windows, העברית מוצגת הפוכה:
- מקלידים: "שלום"
- מוצג: "םולש"

זו בעיה קריטית למי שרוצה לעבוד עם Claude Code בעברית בטרמינל.

---

## הפתרון המומלץ ✅

### Windows Terminal 1.18 - הגרסה האחרונה שבה העברית עובדת מעולה!

אחרי בדיקות מעמיקות (כולל הרצה על סביבות נקיות במכונה וירטואלית + 2 מחשבים פיזיים), נמצא ש:

- ✅ **Windows Terminal 1.18** - עברית עובדת מצוין
- ❌ **Windows Terminal 1.23** - עברית הפוכה (בעיה ידועה)

### מדריך התקנה מלא

קישור למדריך המפורט:
**https://win-terminal-hebrew-118.base44.app/**

המדריך כולל:
1. הוראות התקנה של גרסה 1.18
2. מניעת עדכון אוטומטי ל-1.23
3. בדיקת אבטחה מקיפה (CVE validation)
4. טיפים ופתרונות לבעיות נפוצות

---

## למה גרסה 1.18?

### בדיקת אבטחה ✅
בוצעה בדיקה מקיפה מול מאגרי CVE:
- ✅ גרסה 1.18 **בטוחה לשימוש**
- ✅ אין פגיעויות ביטחון ידועות
- ✅ מתאימה לסביבות פרודקשן

### יציבות ✅
- נבדקה על 2 מחשבים פיזיים
- נבדקה במכונה וירטואלית נקייה
- עובדת באופן עקבי ויציב

---

## אלטרנטיבות שלא עבדו ❌

נוסו הפתרונות הבאים **ללא הצלחה** ב-Windows Terminal 1.23:

1. ❌ שינויי JSON בהגדרות Terminal
2. ❌ הגדרות Unicode במערכת ההפעלה
3. ❌ פונטים מיוחדים (Consolas, Cascadia, etc.)
4. ❌ כלי AI שונים (כולל Claude עצמו)
5. ❌ Locale/Region changes
6. ❌ PowerShell encoding settings

**שום שינוי לא עזר ב-1.23!**
